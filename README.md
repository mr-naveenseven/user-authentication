## ğŸ” Authentication Flow

1. User signs up via `/auth/signup`
2. User logs in via `/auth/login`
3. Server issues a JWT token
4. Client includes the token in all requests to `/user` endpoints as:

```makefile
Authorization: Bearer <token>
```
